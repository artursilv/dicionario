DICIONÁRIO

  • API (Application progamming Interface): é um cojunto de rotinas e padrões estabelecidos por um software para a utiização das suas funcionalidades por aplicativos que não pretendem envolver-seem detalhes de implementação do software, mas apenas usar seus serviços.
  • CLI (Command Line Interface): meio de interagir com um progama de computador, onde o utilizador emite comandos para o programa sob a forma de sucessivas linhas de texto.
  • CRUD (Creat Read Update Delte): são as quatro operações básicas (criação, consulta, atualização e destruição de dados) utilizadas em bases de dados relacionais.
  • CMS (Content Management System): sistema online que permite colocar um site no ar de forma prática e rápida.
  • DNS (Domian Name System): é um sistema hierárquico e distribuido de gestão de nomes para computadores, serviços ou qualuqer máquina conectada á internet ou a uma rede privada. Faz a associação entre várias informações atribuídas a nomes de domínios e cada entidade participante.
  • DOM (Document Object Model): é uma convenção multiplataforma e independente de lingaugem de programação, fiscalizada pelo entidade World Wide Web Consortium (W3C), para representção e interação com objetos em documentos HTML, XHTML e XML.
  • ES (ECMAScript): é uma lingaugem de porgramação, baseada em scripts, padronizadapela ECMA Internacional na especialização ECMA-262 e ISO/IEC 16262. Esta linguagem é usada em tecnologias para internet para a criação de scripts executados no cliente e no servidor(Node JS), sendo inspirada na linguagem de programação JavaScript, além de permitir outras implemnetações independentes.
  • HTTP (HyperText Transfer Protocol): é um protocolo de comunicação utilizado para sistemas de informação de hipermídia, distribuídos e colaborativos.
  • IDE (Integrated Development): é um programa de computador que reúne características e ferramentas de apoio ao desenvolvimento de software com o objetivo de agilizar este processo.
  • IP (Internet Protocol): é um número identificador dado ao seu computador, ou roteador, ao conectar-se à rede.
  • JSON (JavaScript Object Notation): é um formato compacto, de padrão aberto independente, de troca de dados simples e rápida entre sistemas, especificado por Douglas Crockford em 2000, que utiliza texto legível a humanos, no formato atributo-valor.
  • MVC (Model Viem Controller): é um padrão de arquitetura de software responsável por contribuir na otimização da velocidade entre as requisições feitas pelo comando dos usuários.
  • OOP/POO (Object Oriented Progamming): é um paradigma de programação que se propõe a abordar o design de um sistema em termos de entidades, os objetos, e relacionamentos entre essas entidades.
  • PWA (Progressive Web App): é um termo usado para denotar um conjunto de processos de desenvolvimento de software.
  • REGEX (Regular Expression): forma concisa e flexível de indentificar cadeias de caracteres de interesse, como caracteres particulares, palavras ou padrões de caracteres.
  • REST (Representation State Transfer): é um estilo de arquitetura de software que define um conjunto de restrições a serem usadas para a criação de web services.
  • SEO (Search Engime Optimizaiton): é o conjunto de técnicas usadas, geralmente divididas entre tecnologia, conteúdo e autoridade, para alcançar bom posicionamento de páginas de um site no Google e em outros buscadores, gerando tráfego orgânico.
  • SDK (Software Development Kit): são ferramentas fornecidas pelo fabricante de uma plataforma de hardware, sistema operacional ou linguagem de programação.
  • UI (User Interface): corresponde ao layout de uma aplicação, ou seja, aos elementos gráficos que aparecem em uma página da web, um aplicativo ou um software.
  • UX (User Experience): é uma prática que trabalha a interdisciplinaridade, abordando desde branding e design até redação e tecnologia, os quais todos estão interligados.
  • WWW (World Wide Web): designa um sistema de documentos em hipermídia que são interligados e executados na Internet.
  • XML(Extensible Markup Language): é uma recomendação da W3C para gerar linguagens de marcação para necessidades especiais.
